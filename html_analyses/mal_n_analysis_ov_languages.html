MAL_n Analysis: OV Languages (VO Score < 33%)

Head-final/OV languages where the verb typically follows its object.

⚙️ Statistical Threshold: MIN_COUNT = 100

A configuration (specific n value) is only included in regression calculations and averages if it has at least 100 occurrences.

Configurations with fewer occurrences are shown in grey for reference but are not used for computing slopes (β) or decrease ratios.

Data source: Bilateral dependency configurations from Universal Dependencies treebanks

MAL_n: Arithmetic mean of constituent sizes for heads with n total dependents

Log-Log Regression (β = MAL Effect Score)

Each language has an inline plot showing log(n) vs log(MAL_n) with linear regression:

Regression 2→max: Regression from n=2 to max where count ≥ 100. MAL_1 is excluded because it often behaves atypically and can distort the regression slope.

β (slope): The negative of the log-log regression slope. Under MAL: log(MAL_n) = a - β·log(n)

• β > 0.1 (green): MAL compliance

• β < -0.1 (red): Anti-MAL

• |β| ≤ 0.1 (yellow): Weak effect

Local Change Scores (n→n+1)

Score: [ln(MAL_n) - ln(MAL_{n+1})] / [ln(n+1) - ln(n)]

Only computed between consecutive n values that both have count ≥ 100

Visualizations

1. Individual Normalized MAL Curves in Log-Log Space (MAL_n / MAL_2)

Interpretation: Each line represents one language's MAL trajectory normalized by its MAL_2 value, plotted in log-log space: x = log(n), y = log(MAL_n / MAL_2). This removes differences in absolute constituent size and allows direct comparison of the rate of decline across languages.

Key insight: All curves pass through the anchor point (log 2, 0) since MAL_2/MAL_2 = 1 and log(1) = 0. The thick black line shows the cross-linguistic mean. n=1 is included: languages where MAL_1 > MAL_2 appear above the anchor; languages where MAL_1 < MAL_2 appear below. Hover over lines to see language names.

2. Data Availability by Number of Dependents

Interpretation: This chart shows how many languages have sufficient data (≥100 occurrences) at each value of n (up to n=5). The number above each bar indicates the language count.

3. Individual Language MAL_n Trajectories

Interpretation: Each line represents one language's MAL_n trajectory. The thick black line shows the cross-linguistic mean. Most languages show a downward trend (MAL compliance), but there is considerable variation in both the starting point (MAL_1) and the slope of decline.

Interactive: Hover over lines to see language names.

4. Histogram of All Local Change Scores

Interpretation: This histogram pools all local change scores (MAL_n − MAL_{n+1}) / MAL_n across all languages and all transitions n→n+1 together. The x-axis labels show the center of each 0.2-wide bin. A distribution centered above zero indicates that, on average, constituent size decreases with n (MAL compliance).

Important caveat: Because the histogram mixes transitions from different values of n, it can be misleading. Transitions at higher n (e.g., 4→5 or 5→6) are based on far fewer languages and noisier data, so their contribution fluctuates. This chart gives a rough first impression of the overall direction of the MAL effect, but the regression slope β(2→max) shown in Charts 6 and 7 is a much more reliable measure: it captures the overall trend per language rather than individual transition-by-transition fluctuations.

5. −β(2→max) vs. β(1→2) — Regression Slope vs. Initial Transition

Interpretation: Each point represents one language. The x-axis shows β(1→2) — the local change score for the first transition (n=1 to n=2). The y-axis shows −β(2→max) — the negated log-log regression slope from n=2 onward. Both axes use the same sign convention: positive = MAL compliance (constituent size decreases with n). MAL-compliant languages cluster in the top-right quadrant.

Key insight: A positive correlation between these two measures would suggest that languages with a strong initial MAL effect (large β(1→2)) also maintain a steep decline for higher n (large −β(2→max)). A lack of correlation might indicate that the initial transition is governed by different factors than the subsequent ones.

Color coding: Points are colored by language family. Hover for details.

5b. −β(2→max) vs. −β(1→max) — Regression Slopes With and Without MAL_1

Interpretation: Each point represents one language. The x-axis shows −β(1→max) — the negated log-log regression slope computed from n=1 (including MAL_1). The y-axis shows −β(2→max) — the negated log-log regression slope from n=2 (excluding the often atypical MAL_1). Both axes use the same sign convention: positive = MAL compliance.

Key insight: Points near the diagonal (y = x) indicate languages where including or excluding MAL_1 makes little difference. Points well above the diagonal have a stronger MAL effect when MAL_1 is excluded (MAL_1 is unusually large, pulling the slope toward zero). Points well below the diagonal have a weaker MAL effect when MAL_1 is excluded (MAL_1 is unusually small or MAL_1 contributes positively to the slope).

Color coding: Points are colored by language family. Hover for details.

6. MAL Effect −β(2→max) by Language Family

Interpretation: This chart shows −β(2→max), the negated log-log regression slope from n=2 onward. Positive values = MAL compliance (constituent size decreases with n); negative values = anti-MAL behavior. This sign convention matches the main table and Chart 5.

Family comparison: This chart groups languages by family to reveal whether MAL strength varies systematically across genealogical groups. Individual language dots are shown with jitter.

Key insight: If MAL is a universal constraint, we expect positive values across all families. Systematic differences might indicate structural factors that modulate MAL strength.

6b. MAL Effect −β(2→max): Indo-European vs Non-Indo-European

Interpretation: This chart compares −β(2→max) between Indo-European and non-Indo-European languages. Positive values = MAL compliance.

Key question: Is MAL compliance an artifact of over-representing Indo-European languages in the sample, or does it hold equally well for non-Indo-European languages?

6c. MAL Effect −β(2→max): OV vs NDO vs VO

Interpretation: This chart compares −β(2→max) across word-order typological groups based on WALS VO scores: OV (score < 0.33), NDO/mixed (0.33–0.66), VO (score > 0.66). Positive values = MAL compliance.

Key question: Does basic word order (head-initial vs head-final) modulate MAL strength? OV languages place the head last, which may create different dependency-length pressures than VO languages.

7. World Map: Geographic Distribution of MAL Effect −β(2→max)

Interpretation: Each dot represents a language positioned at its geographic location. The color indicates −β(2→max), the negated log-log regression slope from n=2. Positive values = MAL compliance (constituent size decreases with n).

Color scale: Green = strong MAL compliance (positive −β, constituent size decreases with n), Yellow = weak/neutral effect, Red = anti-MAL (negative −β, constituent size increases with n).

Geographic patterns: If MAL is truly universal, we expect green dots distributed across all continents. Clustering of colors might suggest areal effects or shared structural features.

Note: Only languages with geographic coordinates available in WALS are shown.

8. R² Goodness-of-Fit Distribution for β(2→max) Regressions

Interpretation: This histogram shows the distribution of R² values from the log-log regressions that produce each language's β(2→max) slope. R² measures how well a straight line in log-log space fits a language's MAL_n trajectory — in other words, how closely the relationship between log(n) and log(MAL_n) follows the power-law form predicted by Menzerath-Altmann's Law.

Note on color: The blue shading here indicates regression fit quality, not MAL compliance direction. Darker blue = better fit (higher R²), lighter blue = poorer fit (lower R²). This is deliberately distinct from the green/red color scheme used in Charts 6–7, which encodes the sign of β.

High R² (close to 1.0): The log-log regression captures nearly all the variance in the data, meaning the MAL_n values follow a regular power-law decay. This indicates that the Menzerath-Altmann relationship is well-described by a single regression slope β for that language.

Low R² (close to 0): The log-log regression is a poor fit, suggesting that the language's MAL_n trajectory does not follow a simple power-law pattern. This could indicate non-monotonic behavior, structural noise, or insufficient data at certain n values.

Key insight: If most languages cluster at high R² values, it supports the universality of the power-law form of MAL. A bimodal or dispersed distribution would suggest that MAL operates differently across language types, and that the single-slope summary β may be misleading for some languages.

Threshold line: The dashed vertical line at R²=0.90 marks a conventional threshold for a "good" fit. Languages above this threshold have highly predictable MAL trajectories.

9. Negated Mean Local Change Scores by Language Family × Transition

Interpretation: This heatmap shows the negated mean local change score for each combination of language family (rows) and transition (columns: 1→2, 2→3, 3→4, 4→5). The original local change score at transition n→n+1 is (MAL_n − MAL_{n+1}) / MAL_n; here it is negated so that negative values = MAL compliance, consistent with the β slope convention used throughout this report.

Color coding: Green cells indicate negative values (MAL-compliant: constituent size decreases with more dependents), red cells indicate positive values (anti-MAL: size increases), and white/near-white cells indicate near-zero change. This matches the green/red convention of Charts 6 and 7 where negative β = MAL compliance.

Key insight: This chart reveals whether the MAL effect is uniform across all transitions within a family, or whether certain transitions are more "critical" than others. For instance, if the 1→2 transition consistently shows the most negative values across families, this would suggest that the compression of constituents is most pronounced when going from one to two dependents.

Family variation: Rows are sorted by the overall mean (most negative = strongest MAL compliance at top). Families at the top show the strongest MAL compliance across transitions, while those at the bottom may show weaker or anti-MAL patterns. Large differences between columns within a row indicate that MAL strength is transition-dependent for that family.

Cell annotations: Each cell shows the mean score and the number of languages (n=...) contributing to that mean. Cells with very few languages should be interpreted with caution.
