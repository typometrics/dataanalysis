MAL_n Analysis: OV Languages (VO Score < 33%)

Head-final/OV languages where the verb typically follows its object.

⚙️ Statistical Threshold: MIN_COUNT = 100

A configuration (specific n value) is only included in regression calculations and averages if it has at least 100 occurrences.

Configurations with fewer occurrences are shown in grey for reference but are not used for computing slopes (β) or decrease ratios.

Data source: Bilateral dependency configurations from Universal Dependencies treebanks

MAL_n: Arithmetic mean of constituent sizes for heads with n total dependents

Log-Log Regression (β = MAL Effect Score)

Each language has two inline plots showing log(n) vs log(MAL_n) with linear regression:

Regression 1→max: Uses all available data points (n=1 to max) where count ≥ 100

Regression 2→max: Excludes n=1 (starts from n=2 to max) where count ≥ 100

β (slope): The negative of the log-log regression slope. Under MAL: log(MAL_n) = a - β·log(n)

• β > 0.1 (green): MAL compliance

• β < -0.1 (red): Anti-MAL

• |β| ≤ 0.1 (yellow): Weak effect

Local Change Scores (n→n+1)

Score: [ln(MAL_n) - ln(MAL_{n+1})] / [ln(n+1) - ln(n)]

Only computed between consecutive n values that both have count ≥ 100

Visualizations

1. Normalized MAL Curve in Log-Log Space (MAL_n / MAL_2)

Interpretation: This chart shows the relative change in constituent size in log-log space, normalized by each language's MAL_2 value. This allows direct comparison of the rate of decline across languages, independent of their initial constituent sizes.

Key insight: All languages start at 1.0 for n=2; the steeper the decline, the stronger the MAL effect. Only data up to n=5 is shown, as higher n values have insufficient language coverage.

2. Data Availability by Number of Dependents

Interpretation: This chart shows how many languages have sufficient data (≥100 occurrences) at each value of n (up to n=5). The number above each bar indicates the language count.

3. Individual Language MAL_n Trajectories

Interpretation: Each line represents one language's MAL_n trajectory. The thick black line shows the cross-linguistic mean. Most languages show a downward trend (MAL compliance), but there is considerable variation in both the starting point (MAL_1) and the slope of decline.

Interactive: Hover over lines to see language names.

4. Histogram of All Local Change Scores

Interpretation: This histogram shows the overall distribution of local change scores across all languages and transitions. A distribution centered above zero indicates general MAL compliance. The spread shows how consistent the MAL effect is across the dataset.

5. β(2→max) vs. β(1→2) — Regression Slope vs. Initial Transition

Interpretation: Each point represents one language. The x-axis shows β(1→2) — the local change score for the first transition (n=1 to n=2). The y-axis shows β(2→max) — the log-log regression slope from n=2 onward, capturing the overall rate of decline.

Key insight: A correlation between these two measures would suggest that languages with a strong initial MAL effect also maintain a steep decline for higher n. A lack of correlation might indicate that the initial transition is governed by different factors than the subsequent ones.

Color coding: Points are colored by language family. Hover for details.

6. MAL Effect β(1→max) by Language Family

Interpretation: The MAL effect β(1→max) is the slope of the log-log regression of MAL_n on n, starting from n=1. Negative values indicate MAL compliance (constituent size decreases with n); positive values indicate anti-MAL behavior.

Family comparison: This chart groups languages by family to reveal whether MAL strength varies systematically across genealogical groups. Box plots show the distribution within each family.

Key insight: If MAL is a universal constraint, we expect negative slopes across all families. Systematic differences might indicate structural factors that modulate MAL strength.

7. World Map: Geographic Distribution of MAL Effect β(1→max)

Interpretation: Each dot represents a language positioned at its geographic location. The color indicates the MAL effect β(1→max) — the log-log regression slope from n=1.

Color scale: Green = strong MAL compliance (negative slope, constituent size decreases with n), Yellow = weak/neutral effect, Red = anti-MAL (positive slope, constituent size increases with n).

Geographic patterns: If MAL is truly universal, we expect green dots distributed across all continents. Clustering of colors might suggest areal effects or shared structural features.

Note: Only languages with geographic coordinates available in WALS are shown.

8. R² Goodness-of-Fit Distribution for β(1→max) Regressions

Interpretation: This histogram shows the distribution of R² values from the log-log regressions that produce each language's β(1→max) slope. R² measures how well a straight line in log-log space fits a language's MAL_n trajectory — in other words, how closely the relationship between log(n) and log(MAL_n) follows the power-law form predicted by Menzerath-Altmann's Law.

Note on color: The blue shading here indicates regression fit quality, not MAL compliance direction. Darker blue = better fit (higher R²), lighter blue = poorer fit (lower R²). This is deliberately distinct from the green/red color scheme used in Charts 6–7, which encodes the sign of β.

High R² (close to 1.0): The log-log regression captures nearly all the variance in the data, meaning the MAL_n values follow a regular power-law decay. This indicates that the Menzerath-Altmann relationship is well-described by a single regression slope β for that language.

Low R² (close to 0): The log-log regression is a poor fit, suggesting that the language's MAL_n trajectory does not follow a simple power-law pattern. This could indicate non-monotonic behavior, structural noise, or insufficient data at certain n values.

Key insight: If most languages cluster at high R² values, it supports the universality of the power-law form of MAL. A bimodal or dispersed distribution would suggest that MAL operates differently across language types, and that the single-slope summary β may be misleading for some languages.

Threshold line: The dashed vertical line at R²=0.90 marks a conventional threshold for a "good" fit. Languages above this threshold have highly predictable MAL trajectories.

9. Negated Mean Local Change Scores by Language Family × Transition

Interpretation: This heatmap shows the negated mean local change score for each combination of language family (rows) and transition (columns: 1→2, 2→3, 3→4, 4→5). The original local change score at transition n→n+1 is (MAL_n − MAL_{n+1}) / MAL_n; here it is negated so that negative values = MAL compliance, consistent with the β slope convention used throughout this report.

Color coding: Green cells indicate negative values (MAL-compliant: constituent size decreases with more dependents), red cells indicate positive values (anti-MAL: size increases), and white/near-white cells indicate near-zero change. This matches the green/red convention of Charts 6 and 7 where negative β = MAL compliance.

Key insight: This chart reveals whether the MAL effect is uniform across all transitions within a family, or whether certain transitions are more "critical" than others. For instance, if the 1→2 transition consistently shows the most negative values across families, this would suggest that the compression of constituents is most pronounced when going from one to two dependents.

Family variation: Rows are sorted by the overall mean (most negative = strongest MAL compliance at top). Families at the top show the strongest MAL compliance across transitions, while those at the bottom may show weaker or anti-MAL patterns. Large differences between columns within a row indicate that MAL strength is transition-dependent for that family.

Cell annotations: Each cell shows the mean score and the number of languages (n=...) contributing to that mean. Cells with very few languages should be interpreted with caution.
